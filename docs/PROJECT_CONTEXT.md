# Jogging Web — Project Context

## Rol

Soy el desarrollador web del proyecto.

## Objetivo del sitio

El sitio web de Jogging debe presentar el estudio, mostrar sus servicios, exhibir portfolio y facilitar el contacto comercial.

## Stack confirmado para V1

* Astro
* TypeScript
* Tailwind CSS
* Sanity
* GitHub
* Vercel

## Servicios definidos

* Social Media Estratégico
* Proyectos 3D, CGI y FOOH
* Ecosistemas Web
* Comunicación Empresarial

## Home

La Home final será una ilustración de escritorio interactiva.

La Home debe ocupar todo el viewport.

La interacción pensada es que algunos objetos puedan reaccionar al hover, focus o tap para acceder a secciones o funciones.

## Decisiones confirmadas

* El botón “MENÚ” debe abrir la navegación.
* El portfolio es importante para vender.
* Hay necesidad comercial de avanzar con una primera versión funcional.
* El Figma de Portfolio tiene el primer frame no aprobado.
* Van todos los casos aprobados del Figma.

## Pendiente de definición

* Contenido del monitor.
* Función real del botón de sonido.
* Si habrá música o audio.
* Qué objetos exactos de la Home serán interactivos.
* Coordenadas finales de hotspots.
* Textos finales.
* Flujo final de contacto.
* Schemas definitivos de Sanity.

## Reglas para desarrollo

* No inventar contenido final.
* No inventar funciones para objetos no confirmados.
* No inventar comportamiento del monitor.
* No implementar audio hasta que esté definido.
* No crear schemas definitivos de Sanity sin validación.
* Usar Tailwind para estilos.
* No usar CSS Modules.
* No usar SCSS.
* No usar Bootstrap.
* No agregar React salvo que sea necesario y esté justificado.
